## How to Use This Document

This is a **living document** for capturing feature ideas throughout the project lifecycle. Ideas flow in from:
- `/brainstorm` sessions (automatically appended)
- Stakeholder feedback
- Implementation discoveries
- Retrospective findings

Ideas here are **raw and unfiltered** — they don't need to be feasible or approved. The `/project-requirements` and `/project-epics` commands pull from this document when defining requirements and stories.

### Priority Guide

| Priority | Meaning |
|----------|---------|
| Must | Core to the product vision — ship is incomplete without it |
| Should | High value, expected by users, plan to include |
| Could | Nice to have if time permits |
| Won't (this version) | Explicitly deferred — captured for future consideration |

### Status Guide

| Status | Meaning |
|--------|---------|
| Idea | Raw concept, not yet evaluated |
| Exploring | Being researched or prototyped |
| Accepted | Moving to PRD/epics |
| Deferred | Parked for a future version |
| Rejected | Evaluated and declined (keep for reference with reason) |
