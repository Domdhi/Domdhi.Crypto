# /evolve — Cycle Rollover (archive done, re-plan from reality)

The third **post-MVP lifecycle** command (Tier 3). `/listen` gathers signals and `/triage` turns them into stories, but both assume a backlog still exists to append to. `/evolve` owns the transition the harness was missing entirely: **the initial backlog is fully delivered — now what?**

It is `/create:new-project` run backwards-then-forwards. Instead of *zero → plan*, it is *completed cycle → archive → re-plan from production evidence → fresh cycle N+1 backlog*. Like `/create:new-project`, it **orchestrates existing commands** (`/listen`, `/review:retro`, `/create:project-brief` → `requirements` → `epics`) rather than reimplementing them — the new logic is the close-out gate, the history-preserving archive, and seeding the re-plan with *evidence* instead of a blank interview.

> **Origin** (`docs/todo/_design-notes.md`, "Post-MVP lifecycle gap"): *"Dispatch finished 107/107 stories and landed in stall state. The handoff said 'pick next epic' but there was no next epic."* `/evolve` is the command that answers that handoff.

## Telemetry (run first)

This command is user-typed, so it does not fire `PostToolUse:Skill` — without this it leaves no `command_invocation` row and fleet analytics under-count human-driven runs. Self-log the invocation before anything else (best-effort — if it fails, continue regardless):

```bash
node .claude/core/telemetry-log.js evolve
```

## Design basis

- **Archive, never delete.** The completed cycle's `_backlog.md` + TODO files move to `docs/todo/_archive/cycle-{N}-{stamp}/` via `git mv` (lineage follows in log/blame) **and** stay browsable on disk, accompanied by a self-contained `_cycle-summary.md`. The core promise is "don't lose the history" — so deletion is never on the table.
- **Evolve, don't restart.** The re-plan is seeded with *reality* — what cycle N actually shipped, what production telemetry is saying (`/listen`), what the retros learned, what got deferred. A blank-slate re-interview would throw away everything the cycle taught. Evidence-first is the difference between `/evolve` and re-running `/create:new-project`.
- **Gate the close-out — reconcile before you force.** You never archive a half-done cycle. But "unchecked" ≠ "unfinished": on an actively-worked repo the usual cause is *trackers left stale after the work shipped*. So the gate is three-way, not binary — when git shows the open stories actually shipped (completion commits + green gate), the right move is **reconcile** the trackers to reality and archive clean, *not* `--force` (which would falsely record shipped work as carried-forward). `--force` stays the explicit override for *genuinely* incomplete work; reconcile is preferred whenever reality contradicts the checkboxes. Reconcile is never offered when git can't corroborate the work.
- **The stack carries forward.** Architecture (`_project-architecture.md`) is **not** regenerated by default — a delivery cycle rarely changes the tech stack. Only brief → requirements → epics re-run. (`--replan-arch` to include it.)
- **Orchestrate, don't reimplement.** Chain the existing planning pipeline, exactly as `/create:new-project` does (brief → requirements → epics → todo → lead-epic make-ready). Interactively, each sub-command commits its own work and `/evolve` commits the archive + summary + evidence digest. **Headless, inlining the chain into fewer commits is equally valid** (adopter finding EV5) — the contract is the *artifacts* (fresh brief/PRD/backlog/master **plus a runnable lead-epic TODO**) plus a committed, history-preserving archive + evidence digest, **not** a specific commit count. Don't reimplement the planning logic itself; reuse the skills the sub-commands load.
- **End runnable, not just planned.** A re-plan that stops at the backlog leaves the user back in the "pick next epic — but there's no contract to run" stall, one rung up. So by default `/evolve` materializes the lead epic into an execution-ready `TODO_epic*.md` (Step 5.6) — but only the lead epic, just-in-time; the rest of the backlog is broken down later as it's reached.

## Variables

SCOPE: $ARGUMENTS

- `--force` — archive even with incomplete (`[ ]`/`[!]`) stories. The incomplete set is recorded in `_cycle-summary.md` under "Unfinished, carried forward" and seeded into the next cycle's backlog — never silently dropped.
- `--dry-run` — report the close-out plan (what would be archived, the evidence digest, the re-plan path) but write **nothing** — no archive, no git mv, no sub-command invocation, no commit. (Sole exception: the one-line telemetry log at the top still fires — the command *was* invoked. It writes only a `command_invocation` row, never project state.)
- `--replan-arch` — also regenerate `_project-architecture.md` (default: architecture carries forward unchanged).
- `--no-todo` — skip Step 5.6 (the lead-epic make-ready). Default is **on**: `/evolve` ends with the first epic's execution-ready `TODO_epic*.md` so the cycle is *runnable*, not just *planned*. Pass `--no-todo` to stop at the backlog (e.g. you want to hand-pick the first epic, or re-prioritize before breaking anything down).
- A trailing free-text **cycle label** — a friendly name folded into the archive dir + summary (e.g. `/evolve "MVP launch"` → `cycle-1-260606-mvp-launch/`). Optional.

## Output

- `docs/todo/_archive/cycle-{N}-{stamp}[-{label}]/` — the archived `_backlog.md`, `TODO_{Project}.md`, `TODO_epic*.md`, the cycle's `_project-brief.md` + `_project-requirements.md` snapshots, and `_cycle-summary.md` (the self-contained close-out record).
- `docs/todo/_backlog.md` + `docs/TODO_{Project}.md` — **fresh**, for cycle N+1 (regenerated by the chained pipeline).
- `docs/_project-brief.md` + `docs/_project-requirements.md` — regenerated for cycle N+1 (old copies live in the archive).
- `docs/todo/TODO_epic{NN}_{slug}.md` — the lead epic's **execution-ready** checklist (Step 5.6, unless `--no-todo`), so the cycle ends runnable. This is the file `/run-todo` / `/do` actually consume.
- `docs/.output/work/{stamp}/evolve-evidence.md` — the synthesized evidence digest that seeded the re-plan.

`{stamp}` is a single `date +%y%m%d-%H%M` run-stamp, computed once at the start of the run and reused verbatim everywhere below (per the run-stamp convention in CLAUDE.md), so a same-day re-run never collides.

## Workflow

### Step 1: Gate — is the cycle closeable?

Compute the run-stamp once: `STAMP=$(date +%y%m%d-%H%M)`. Reuse it for every path below.

- Detect the project name (from the `docs/TODO_*.md` filename, else `_project-context.md`, else the repo basename).
- Read `docs/todo/_backlog.md`, `docs/TODO_{Project}.md`, and every `docs/todo/TODO_epic*.md`.
- Tally story checkboxes across all of them: `[x]` done, `[ ]` open, `[~]` deferred, `[!]` blocked. **Checkboxes are authoritative for the gate decision.**
- **Emoji cross-check (EV4 — flag disagreement, don't silently trust one view).** `_backlog.md` and `TODO_{Project}.md` often carry an *emoji* Story-Index in parallel (✅ done / ⬜ todo / 🔲 etc.) alongside (or instead of) checkboxes. After the checkbox tally, scan those emoji status columns and compare per story:
  - If the emoji view and the checkbox view **disagree** for any story (e.g. checkbox `[x]` but Story-Index row says `⬜`, or vice-versa), **print a ⚠ drift warning** naming the file + line + both values. This is internal backlog drift (a stale representation), and the checkbox-only tally is blind to it — so surface it rather than archiving on a half-stale backlog.
  - Disagreement does **not** by itself block the archive (checkboxes win), **but** if the emoji view would have made the cycle look *incomplete* while checkboxes say done, say so explicitly in the report so the human can confirm the code really shipped before the cycle is sealed.
  - If a file uses only emoji (no checkboxes at all), tally from the emoji instead and note that the gate fell back to the emoji view for that file. **For these emoji-only files the emoji ARE the gate signal:** map ⬜/🔲 (and any non-✅ todo-state) to the same gate weight as `[ ]`, so an emoji-only file that is genuinely incomplete blocks the archive exactly like an unchecked checkbox would. (Without this the checkbox-only gate is blind on emoji-only files — it sees zero `[ ]`/`[!]` and archives an incomplete cycle. EV4 gate hole.)
- **Gate decision — three-way (reconcile ▸ stop ▸ force), NOT a binary stop/force.** If any `[ ]`/`[!]` story (or non-✅ todo-state on an emoji-only file) exists and `--force` is NOT set, do **not** STOP reflexively. The most common cause of open checkboxes on an actively-worked repo is *trackers left stale after the work already shipped* — the checkboxes are wrong, not the cycle. So first cross-check the open set against **git/gate reality**, then branch:
  - **Reality check.** For each open/blocked story (or its epic), look for evidence it actually shipped: a completion commit in `git log` referencing the epic/story, the feature present in the code, and the build+test gate currently green (read `docs/.output/telemetry/_latest-summary.json` `overall`, or run `node .claude/core/gate.js test`).
  - **① Reconcile (preferred when reality says DONE).** If the open stories map to epic-complete commits **and** the gate is green, the trackers are *stale*, not the work — and `--force` would be **wrong** here: it records shipped work as "Unfinished, carried forward" and pollutes the next cycle's backlog. Instead **confirm with the user via `AskUserQuestion`**, e.g. *"git shows epics {X,Y} shipped and the gate is green, but {K} checkboxes are still `[ ]`. Reconcile the trackers to git reality, then archive?"* On confirm: rewrite the stale `[ ]`→`[x]` (and emoji ⬜/🔲→✅) **only for the stories git corroborates**, commit that reconciliation on its own (`docs: /evolve — reconcile cycle {N} trackers to git reality before archive`), **then proceed** with the archive as a clean, complete cycle. This is the resolution EV4 stops one step short of: EV4 *detects* the drift; this *resolves* it correctly. (Captured as memory `evolve-reconcile-tracker-drift-before-archive`.)
  - **② Stop (work genuinely incomplete).** If git does **not** corroborate the open stories — no completion commits, feature absent, or gate red — the trackers are right and the cycle is genuinely unfinished. **STOP**, print the incomplete stories (file + line), and:

    ```
    Cycle not complete — {K} stories still open/blocked, and git does not show them shipped.
    Finish them (/do, /run-todo), or re-run with --force to archive anyway (incomplete items carry forward).
    ```
  - **③ Force (explicit override).** `--force` archives regardless and records the incomplete set under "Unfinished, carried forward" in `_cycle-summary.md` — never silently dropped. Use it only to *deliberately* carry genuinely-incomplete work forward, **not** to paper over stale trackers (that case is ①, reconcile).
  - Per CLAUDE.md gate posture (F3): a gate means *resolve the prerequisite, then resume* — and reconcile ① **is** that resolution when the work is done-but-untracked. `--force` is never the recommended path, and reconcile must **never** be offered when git can't corroborate the work — that would make it a silent `--force` that seals an incomplete cycle.
- Determine the cycle number **N** by the **highest existing cycle number + 1**, not a raw directory count: parse the numeric prefix of each `docs/todo/_archive/cycle-{NN}-*` dir (regex `^cycle-(\d+)-`) and take `max(NN) + 1`. (A raw count miscomputes N if `_archive/` holds a stray, partial, or hand-made dir — e.g. a previous aborted run — producing a skipped or duplicated cycle number.) If `_archive/` is absent or has no `cycle-\d+-` dirs, this is cycle 1 (the original `/create:new-project` backlog).
- State the resolved cycle number, project name, and completion tally in the report.

### Step 2: Gather re-planning evidence (the "from reality" inputs)

This is what makes it *evolve* and not *restart*. Collect, then synthesize into the evidence digest:

- **2a. Production signals.** Glob `docs/.output/intake/*.md` and take the newest. **Staleness is coverage-based, NOT mtime-based** — an intake is fresh only if it actually covers *this* cycle's work, so a same-day file that predates the whole cycle is still stale (Crypto finding: the only intake was same-day but from the Epic-11 era — generated before the cycle it was meant to inform, yet a date-only test read it as fresh). Test it against the cycle's **commit range**, not the file's modification time:
  - Establish the cycle window: start = the most recent `_archive/cycle-*` date (or the cycle's first commit if this is cycle 1); end = `HEAD`.
  - Read the intake's own `generated:` timestamp from its frontmatter (fall back to file mtime only if absent). The intake is **stale** if it was generated *before* a meaningful share of this cycle's work landed — i.e. `git log --oneline --since="<intake generated>"` still shows substantive feature commits from the cycle (not just the chore/handoff/intake commits that followed it). Equivalently: an intake that predates the cycle's first feature commit is stale no matter how recent its date.
  - If stale (or missing), **run `/listen` first** to get a fresh sweep — **but NOT under `--dry-run`** (`/listen` writes + commits an intake file, which would violate the "writes nothing" dry-run contract; under `--dry-run` read the existing intake only and report "(would refresh stale intake — predates the cycle's commit range)"). Read the newest intake.
- **2b. Lessons.** Read every `docs/.output/reviews/retro-*.md` (epic retrospectives from this cycle). If none exist, note it — a cycle with no retros re-plans on thinner evidence.
- **2c. Deferred / waiting work.** Read the triage ledger `docs/.output/triage/_decisions.md` (deferred items whose revisit window has now arrived) and `docs/todo/_feature-ideas.md`. These are candidate cycle N+1 stories.
- **2d. What shipped.** Extract the cycle's delivered scope — the `[x]` stories from the about-to-be-archived backlog, plus the headline commits (`git log` since the cycle's first commit / the prior archive date).
- **2e. Session decisions & pivots (the richest source on an actively-worked repo).** Read the **memory store** — `node .claude/core/memory-manager.js report` and `search` on the project's themes — surfacing recent high-importance `decisions`, `patterns`, and `workflows` memories. Also scan the newest `docs/.output/memories/daily/*.md` for in-session pivots **not yet promoted** to a memory or to `_feature-ideas.md`. This is where a mid-cycle direction change lives (e.g. "we pivoted the product to X") — and on any repo where real work happened in-session it is the **single highest-signal re-planning input**. Skipping it is the one thing that most undermines *evolve, don't restart* (adopter finding EV3, 2026-06-06): a vision pivot recorded only in memory would otherwise be invisible to the re-plan. The store is local-only (gitignored), so this evidence never travels in git — it MUST be read here, at re-plan time, or it is lost.
- **Synthesize** all of the above into the close-out narrative used in Step 4's summary and Step 5's seed. **If a session pivot or decision surfaced in 2e contradicts or extends the shipped vision, name it explicitly in the digest** — the cycle N+1 brief must honor it, not silently re-derive cycle N's direction.

### Step 3: Archive the completed cycle (history-preserving) — skip all writes if `--dry-run`

- `mkdir -p docs/todo/_archive/cycle-{N}-{STAMP}[-{label-slug}]/`
- Move the cycle's files in with **`git mv`** (preserves history; log/blame follow the move). **Move each file individually with a per-file existence guard — never pass a bare `TODO_epic*.md` glob to `git mv`.** A `--no-todo` cycle (or one whose epics were never materialized) has zero `TODO_epic*.md` files; an unexpanded literal glob makes `git mv` error **atomically**, so `_backlog.md` and `TODO_{Project}.md` are left un-moved — and then Step 5 regenerates `_backlog.md` in place, **silently overwriting the un-archived backlog (data loss — the exact opposite of this command's reason to exist)**. Guard each path; fall back to plain `mv` for an untracked file:
  ```bash
  ARCH=docs/todo/_archive/cycle-{N}-{STAMP}
  for f in docs/todo/_backlog.md docs/TODO_{Project}.md docs/todo/TODO_epic*.md; do
    [ -e "$f" ] || continue                      # skip globs that didn't expand
    git mv "$f" "$ARCH/" 2>/dev/null || mv "$f" "$ARCH/"
  done
  ```
- **Assert the archive is populated before Step 5 runs.** After the loop, confirm `_backlog.md` is now inside `$ARCH/` (and absent from `docs/todo/`). If it is not, **STOP** — do not let Step 5 regenerate over an un-archived backlog.
- **Snapshot the cycle's planning context** into the archive too (copy, not move — the create-chain will overwrite the live ones in Step 5). Guard each `cp` so a missing/stub doc cannot abort the step mid-archive:
  ```bash
  for f in docs/_project-brief.md docs/_project-requirements.md; do
    [ -e "$f" ] && cp "$f" "$ARCH/"
  done
  ```
  (These two are content copies, not git-tracked moves — note that in the summary so "history-preserving" isn't over-claimed for them.)
- **Do NOT archive `_feature-ideas.md`** — it is the deferred-ideas bridge between cycles and carries forward into the next backlog.

### Step 4: Write `_cycle-summary.md` (the self-contained close-out record) — skip if `--dry-run`

Into the archive dir, so the folder alone tells the whole story of cycle N:

```markdown
# Cycle {N} Summary — {Project} {label}

**Closed:** {YYYY-MM-DD}  ·  **Span:** {first commit date} → {today}  ·  **Stamp:** {STAMP}
**Completion:** {D} done / {O} open / {DF} deferred / {B} blocked  ({pct}%)

## What shipped
- Epic {NN}: {title} — {n} stories ({headline commits})
- ...

## What production is telling us  (from /listen intake {date})
- {top signals, by source}

## Lessons  (from {n} retros)
- {key pattern / what worked / what didn't}

## Carried forward to cycle {N+1}
- Deferred (triage ledger / feature-ideas): {items}
- Unfinished, carried forward (only if --force): {open/blocked stories}
```

### Step 5: Regenerate the plan for cycle N+1 — skip if `--dry-run`

- Write the evidence digest to `docs/.output/work/{STAMP}/evolve-evidence.md` — the Step 2 synthesis formatted as upstream context (vision-relevant signals, validated demand from shipped scope, lessons, carried-forward items).
- Chain the planning pipeline, **each invocation seeded with the evidence digest as context** (exactly how `/create:new-project` passes interview answers down). Because the live planning docs were snapshotted to the archive in Step 3, regeneration is safe:
  1. **`/create:project-brief`** — pass `docs/.output/work/{STAMP}/evolve-evidence.md` as the input context (Context Mode); answer "replace" when it detects the existing brief. Produces the cycle N+1 vision, grounded in what shipped + what production wants.
  2. **`/create:project-requirements`** — delta PRD from the new brief + evidence.
  3. **`/create:project-architecture`** — **skip by default** (stack carries forward). Run only if `--replan-arch`.
  4. **`/create:project-epics`** — fresh `docs/todo/_backlog.md` for cycle N+1, seeded with the carried-forward deferred items as candidate stories.
  5. **`/create:project-todo`** — regenerate the master `docs/TODO_{Project}.md` index.
- Result: the active `docs/todo/` workspace holds a clean cycle N+1 backlog; cycle N lives entirely in `_archive/`.

### Step 5.6: Materialize the lead epic's execution-ready TODO — skip if `--dry-run` or `--no-todo`

Without this step `/evolve` ends one rung short of runnable: it produces a fresh `_backlog.md` + master index, but its own Step-7 `Next:` line points at `/run-todo` / `/do {first story}` — and **neither can run**, because there is no per-epic `TODO_epic*.md` and no `/todo` contract yet. The user lands back in the same "what now?" stall `/evolve` exists to cure, just one rung higher (adopter finding: *evolve should leave the next TODO execution-ready*). Close that gap:

- **Pick the lead epic.** From the fresh cycle N+1 `_backlog.md`, choose the **first dependency-root epic** — the lowest-numbered epic with no unmet dependencies. Only this one is materialized now; later epics' breakdowns will shift as the lead epic is built, so just-in-time is correct for the rest (do **not** pre-break-down the whole backlog). When the lead epic finishes, running `/todo` with **no argument** auto-resolves the next epic from the master index — so the loop self-continues without re-running `/evolve`.
- **Materialize it.** Chain **`/create:project-epics-todo {lead epic}`** to produce `docs/todo/TODO_epic{NN}_{slug}.md` (per-epic story checklist), then **`/todo {lead epic}`** to make it execution-ready (AC, file lists from real codebase research, wave plan) — `/todo` is epic-number aware and reads the epic's stories straight from `_backlog.md`. Seed both with the same evidence digest. (Both write the SAME epic-keyed file `TODO_epic{NN}_{slug}.md` — `/todo` **enriches/overwrites it in place** (epic-keyed files are a deliberate run-stamp exception, so the second pass is meant to replace the first, not collide). If `/create:project-epics-todo`'s output is already execution-ready for this project's conventions, running `/todo` on top is still valid — it adds the research-backed file lists `/run-todo` trusts.)
- **The Step-7 `Next:` line must now point at this real file** — `/run-todo docs/todo/TODO_epic{NN}_{slug}.md` (or `/do {first story id}`) — not at a backlog that has no runnable contract behind it.
- If `--no-todo`, skip all of the above and let Step 7 say the cycle is *planned, not yet broken down* — `Next: /create:project-epics-todo {lead epic}` then `/todo {lead epic}`.

### Step 6: Commit (main agent) — skip if `--dry-run`

The chained sub-commands (Steps 5 + 5.6) commit their own regenerated docs per the Post-Command Commit Convention. **This step commits only what `/evolve` itself produced:** the archive dir (git-mv'd files + snapshots + `_cycle-summary.md`) and the evidence digest.

Follow CLAUDE.md's Post-Command Commit Convention. Stage specifically (never `git add .`):
```bash
git add docs/todo/_archive/cycle-{N}-{STAMP}/ docs/.output/work/{STAMP}/evolve-evidence.md
```
Write the message to `docs/.output/.commit-msg` (Write tool), then `node .claude/core/commit.js` (inline `git commit -m` is blocked):
```
feat: /evolve — cycle {N} archived, cycle {N+1} planned

{one-line: what shipped in N, what seeds N+1}
```

### Step 7: Report

```
## /evolve — cycle {N} → {N+1}

**Project:** {name}  ·  **Cycle {N} completion:** {pct}% ({D}/{total})
**Archive:** docs/todo/_archive/cycle-{N}-{STAMP}/
**Commit:** {hash, or "dry-run — nothing written"}

### Cycle {N} closed
- Shipped: {epic/story headline counts}
- Lessons folded in: {n retros, n intake signals}

### Cycle {N+1} planned
- Brief: {1-line new vision}
- Backlog: {epic count} epics, {story count} stories ({n} carried forward from deferred)
- Architecture: {regenerated | carried forward unchanged}
- Lead epic made ready: {`TODO_epic{NN}_{slug}.md` — {story count} stories | skipped (--no-todo)}

**Next:** `/review:check-readiness` to validate the new cycle, then:
- (default) `/run-todo docs/todo/TODO_epic{NN}_{slug}.md` — the lead epic is execution-ready.
- (if `--no-todo`) `/create:project-epics-todo {lead epic}` then `/todo {lead epic}` to break down the lead epic first.
```

If `--dry-run`, render the same report but mark each write as *(would write)* and end with the literal: `Dry run — nothing archived or regenerated.`

## Anti-Patterns

- **Archiving an incomplete cycle silently.** The gate (Step 1) blocks open/blocked stories. `--force` is an explicit override that *records* the incomplete set in the summary and carries it forward — it never hides it.
- **`--force`-ing done-but-untracked work.** When git shows the open stories actually shipped (completion commits + green gate), `--force` is the *wrong* tool — it records shipped work as "Unfinished, carried forward" and pollutes the next backlog. Reconcile the trackers to git reality first (Step 1 ①), then archive a clean cycle. Reconcile is gated on git corroboration, so it can never silently seal a genuinely-incomplete cycle.
- **Re-planning from a blank slate.** The whole point is evidence-seeded regeneration. If there is genuinely zero evidence (no intake, no retros, no deferred work), say so plainly in the report — do **not** fabricate a v2 vision out of nothing.
- **Deleting instead of archiving.** Always `git mv` (or `cp` for snapshots). `rm` of cycle history is never correct — "don't lose the history" is the command's reason to exist.
- **Archiving `_feature-ideas.md`.** It bridges cycles and carries forward. Only the backlog + TODO files + planning snapshots get archived.
- **Regenerating architecture by reflex.** The stack carries forward unless `--replan-arch`. Re-running it churns ADRs for no reason on most cycles.
- **Breaking down the whole backlog at rollover.** Step 5.6 materializes only the **lead** epic into a runnable TODO. Pre-breaking-down every epic wastes work — later epics shift as the lead one is built; just-in-time breakdown is correct for the rest.
- **Ending "planned" but not runnable.** Without Step 5.6 the report's `Next:` line points at `/run-todo` with no TODO behind it — the exact stall `/evolve` exists to cure. Default to making the lead epic execution-ready; `--no-todo` is the explicit opt-out, not the norm.
- **Chat-only output.** The archive, summary, evidence digest, fresh backlog, and lead-epic TODO are the artifacts — they hit disk before the report (except `--dry-run`, which writes nothing and says so).
